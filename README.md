# 🚀 Zerodha Clone - MERN Stack Trading Platform

## 📌 Overview

The **Zerodha Clone** is a full-stack web application inspired by the popular stock trading platform Zerodha. This project is built using the **MERN stack (MongoDB, Express.js, React.js, Node.js)** and replicates core functionalities of a modern trading dashboard, including portfolio management, order placement, and real-time UI interactions.

This project demonstrates my ability to design and develop a complete full-stack application from scratch, covering frontend, backend, database integration, and deployment-ready architecture.



## 🎯 Objective

The main goal of this project was to:

* Understand and implement full-stack development using MERN
* Build a real-world scalable application
* Learn how frontend and backend communicate via APIs
* Practice project structuring and modular coding
* Gain experience with Git, GitHub, and version control workflows

## 🔄 How I Built This Project

### 1️⃣ Planning Phase

I started by understanding how Zerodha works:

* Landing page for users
* Dashboard for trading
* Data structures like holdings, orders, and positions



### 2️⃣ Frontend Development

I created two main parts:

* **Frontend (Landing Page)**
* **Dashboard (Main Application UI)**

#### Key Steps:

* Built reusable React components
* Designed UI similar to Zerodha
* Used CSS for styling and layout
* Structured components logically inside `src/`



### 3️⃣ Backend Development

I created a backend server using:

* Node.js
* Express.js

#### Key Steps:

* Setup Express server
* Created routes for API endpoints
* Connected backend with MongoDB
* Structured code into models and schemas



### 4️⃣ Database Design

Used MongoDB to store application data.

#### Collections:

* Holdings
* Positions
* Orders

Each collection has its own schema defining structure and validation.



### 5️⃣ API Integration

Connected frontend and backend using REST APIs:

* Fetch data from backend
* Display dynamically on UI

### 6️⃣ Testing

Used Thunder Client to:

* Test API endpoints
* Verify data flow
* Debug issues

```

# Frontend / Dashboard

```
